## 核心教学思路

新手学 Docker 最大的困惑，不是记不住命令，而是不理解镜像和容器到底是什么关系——

**"我做了镜像，为什么还要 docker run？镜像不就是拿来运行的吗？"**

这个问题是 Docker 教学的钥匙。解释清楚它，镜像和容器的分工（模板 vs 实例）就透了，后面的所有概念都是这个核心决策的自然延伸。

## 教学原则

1. **从核心困惑切入**——"为什么不能直接运行镜像？"是理解镜像 vs 容器的最佳入口
2. **先讲虚拟机做对比**——虚拟机镜像=实例绑死，Docker 镜像≠容器，两种不同的设计哲学
3. **区分环境、运行与数据**——镜像管环境，容器管运行，卷管数据，各司其职
