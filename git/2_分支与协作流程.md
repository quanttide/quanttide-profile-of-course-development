# 课时 2：分支与协作流程

## 2.1 分支是什么

分支必须与三区操作**分属不同课时**，混在一起是最大的教学坑。

分布式背景下的分支很好理解：SVN 的分支是贵的（在服务器上拷贝代码），Git 的分支是便宜的（本地指针，创建和切换都不需要联网）。所以 Git 社区鼓励大量用分支。

## 2.2 两种分支模式

- **Feature branch**：每个人从主干拉一条自己的分支，开发完再合并回来。合并前可以做代码审查。
- **主干分支（trunk-based）**：所有人都在同一条主干上开发。这就是我们当前的实践。

Git 给了你选择的权利，两种模式都可以。

## 2.3 主干分支与自动化

主干分支模式高度依赖流程自动化——每一次提交都直接进入主干，需要有配套的 CI/CD、代码审查、自动化测试来保证质量。这也是我们大量造自动化工具的原因。
